# Digital Library Backend

## Порты и ошибка EADDRINUSE

Если порт 5000 занят (другой процесс или системная служба Windows), сервер **автоматически** запустится на порту **5001**. В консоли появится сообщение вида:
`Port 5000 was in use, using 5001 instead.`

**Чтобы фронтенд работал с бэкендом на 5001:** в папке `frontend` создайте файл `.env` и добавьте:
```
VITE_API_URL=http://localhost:5001
```
Перезапустите фронтенд (`npm run dev`).

Либо освободите 5000 и запустите снова: `npm run freeport`, затем `npm run dev`.

## Рекомендуемый порядок работы (seed и фронтенд в реальном времени)

1. **Один раз запустите бэкенд** и оставьте его работать:
   ```bash
   npm run dev
   ```

2. В **другом терминале** при изменении данных в `seed/seedBooks.js` выполняйте:
   ```bash
   npm run seed
   ```
   Сервер при этом **не перезапускайте** — seed только обновляет базу.

3. Откройте фронтенд в браузере. Когда вы вернётесь на вкладку с приложением после запуска seed, список ресурсов обновится автоматически.

Не запускайте `npm run dev` дважды в разных терминалах — второй запуск даст EADDRINUSE.
